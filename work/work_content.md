
#rule
- 항상 docs/rules에 있는 문서를 참조하고, 지난 작업 목록은 commit 기록을 참조하십시오.
- 반드시 작업 완료 후 컴파일 테스트 후 오리진에 반영하십시오.
- 사용자가 work\work_content.md 파일을 읽고 이 파일을 기준으로 작업을 진행하라는 명시적인
요청사항이 있지 않다면 이 파일에 대한 내용을 기준으로 작업을 진행하지 마십시오.
- 이 프로세스는 비대화형 자율 모드로 실행 중입니다. 사용자가 결정을 내릴 수 없는 상태이므로, 귀하가 자율적으로 결정을 내려야 합니다.
- work 및 pipeline 경로에 있는 파일은 절대 수정하면 안됩니다. (CI/CD에서 사용하는 경로이므로 절대 수정을 해서는 안됩니다.)
- 이전 세션에 이어서 작업을 시작하기전에 반드시 context compaction을 진행하고 하십시오.

#task
 - 이 프로젝트는 c++, directx11 기반 게임 엔진입니다.
 - 렌더러 아키텍처를 구성하고 구현하십시오.
 - 게임 엔진에 속하는 하위 기능(예를 들면 렌더러)들은 모듈(컴포넌트) 형태로 작성되어 동작해야합니다.
 이를 위한 아키텍처 및 구조를 구성하고 구현을 하십시오.
 - *.fbx 와 같은 모델링 파일을 로드해서 처리할 수 있는 기능을 구현하십시오.
 - 에디터는 c# 으로 작성하세요. 닷넷 8.0 이상으로 하세요. 에디터는 각 게임 엔진 모듈들을 하나로 통합합니다. 에디터 프로그램에서 각 모듈들의 기능을 통합적으로 사용할 수 있습니다. 예를 들면 언리얼 엔진 및 유니티 엔진처럼 게임 씬에 오브젝트를 배치하고 실행할 수 있습니다.
 - 에디터를 실행하면 각 하위 모듈들이 로드되고 정상적으로 완료되면 편집이 가능한 에디터 프로그램이 정상 시작됩니다.
 - fbx 파일 로드등, 에셋 제작 도구에서 생성된 3d 오브젝트 파일들을 읽고 처리할 수 있어야합니다.
 - 파일 처리 (시리얼라이제이션, 디시리얼라이제이션)등을 지원하십시오.
 - 스켈레탈, 스태틱 오브젝트 기능을 구현하십시오.
 - 애니메이션 시스템 모듈을 구현하십시오.
 - 작업 정리 후 c#/c++ 컴파일 테스트 완료 하고나서, 커밋 후 오리진에 반영하세요.
 - 게임 엔진 기능을 실제로 샘플 형태로 볼 수 있어야하며, samples 폴더 아래에 목적에 맞게 다시 하위 폴더로 나누어서 여러 샘플 프로그램(*.exe)들이 존재해야합니다. 컴파일 및 빌드에 성공하는지도 확인하십시오.
 - 백엔드 및 프론트엔드에서 CLI 기반 명령어셋을 제공해야합니다. 이를 통해 작업된 기능들을 효과적으로 테스트를 수행하십시오.
